## AI Builder Role Contracts

AI workforce agents operating in this family must:

1. **Read before building.** Read the full canonical reading path (README + ENTERPRISE_MODEL_FAMILIES.yaml + relevant MODELS.yaml) before writing a line of code.
2. **Use 4-layer artifacts.** Every module they write must have an explicit AI-readable blueprint (meaning, model, computation, execution binding). The same standard the organism holds its own docs to.
3. **Pass through gates.** Every build output must pass GATE-VALIDATION (does it compile?) and GATE-BUILD-RELEASE (does it respect all 23 laws?).
4. **Never hardcode canonical constants.** Φ, 873, S₀ — all come from kernel.mo. A hardcoded 0.618 in any build artifact is a LAW-10 violation.
5. **Write proof bundles.** Every build task produces: what was built, what tests passed, what gates were evaluated, what was logged to the ANIMA chain.
